Deployment

This guide covers deploying COO-LLM in various environments, from development to production.

Deployment Optionsâ€‹

Quick Startâ€‹

Local Developmentâ€‹

  1. Clone and build:

    git clone https://github.com/coo-llm/coo-llm-main.git
    cd coo-llm
    go build -o coo-llm ./cmd/coo-llm

  2. Configure environment:

    export OPENAI_API_KEY="sk-your-key"
    export GEMINI_API_KEY="your-gemini-key"

  3. Create config:

    # configs/config.yaml
    version: "1.0"
    server:
      listen: ":2906"

    llm_providers:
      - id: "openai-prod"
        type: "openai"
        api_keys: ["${OPENAI_API_KEY}"]
        base_url: "https://api.openai.com"
        model: "gpt-4o"
        pricing:
          input_token_cost: 0.002
          output_token_cost: 0.01
        limits:
          req_per_min: 200
          tokens_per_min: 100000

    api_keys:
      - key: "test-key"
        allowed_providers: ["*"]
        description: "Test key for development"

    model_aliases:
      gpt-4o: openai-prod:gpt-4o

  4. Run:

    ./coo-llm -config configs/config.yaml

  5. Test:

    curl -X POST http://localhost:2906/v1/chat/completions \
      -H "Authorization: Bearer test-key" \
      -d '{"model": "gpt-4o", "messages": [{"role": "user", "content": "Hello"}]}'

Docker Deploymentâ€‹

  1. Build image:

    docker build -t coo-llm:latest .

  2. Prepare config:

    # Create your config.yaml with API keys and settings
    mkdir configs
    # Edit configs/config.yaml

  3. Run container:

    docker run -p 2906:2906 \
      -e OPENAI_API_KEY="sk-your-key" \
      -e GEMINI_API_KEY="your-gemini-key" \
      -v $(pwd)/configs:/app/configs \
      -v $(pwd)/logs:/app/logs \
      khapu2906/coo-llm:latest

Dockerfile details:

  • Multi-stage build with Go 1.23
  • Alpine Linux base image for minimal size
  • Non-root user for security
  • Config mounted from host for flexibility and security

Docker Composeâ€‹

Full stack with Redis:

# docker-compose.yml
version: '3.8'
services:
  coo-llm:
    build: .
    ports:
      - "2906:2906"
    environment:
      - OPENAI_API_KEY=${OPENAI_API_KEY}
      - GEMINI_API_KEY=${GEMINI_API_KEY}
    depends_on:
      - redis
    volumes:
      - ./configs:/app/configs  # Mount config from host
      - ./logs:/app/logs

  redis:
    image: redis:7
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data

volumes:
  redis_data:

Redis-only for development:

# docker-compose.local.yml
version: '3.8'
services:
  redis:
    image: redis:7
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data

volumes:
  redis_data:

Run:

# Full stack
docker-compose up -d

# Redis only (for local development)
docker-compose -f docker-compose.local.yml up -d

Production Deploymentâ€‹

Environment Variablesâ€‹

Required:

export OPENAI_API_KEY="sk-your-key"
export GEMINI_API_KEY="your-gemini-key"  # Optional
export CLAUDE_API_KEY="your-claude-key"  # Optional

Optional:

export REDIS_ADDR="redis:6379"           # For Redis storage
export REDIS_PASSWORD="your-password"    # If Redis requires auth
export ADMIN_API_KEY="your-admin-key"    # For admin endpoints (future)

Kubernetesâ€‹

Basic Deployment:

apiVersion: apps/v1
kind: Deployment
metadata:
  name: coo-llm
spec:
  replicas: 2
  selector:
    matchLabels:
      app: coo-llm
  template:
    metadata:
      labels:
        app: coo-llm
    spec:
      containers:
      - name: coo-llm
        image: coo-llm:latest
        ports:
        - containerPort: 2906
        env:
        - name: OPENAI_API_KEY
          valueFrom:
            secretKeyRef:
              name: llm-secrets
              key: openai-key
        - name: REDIS_ADDR
          value: "redis-service:6379"
        volumeMounts:
        - name: config
          mountPath: /app/configs
      volumes:
      - name: config
        configMap:
          name: coo-llm-config

Service:

apiVersion: v1
kind: Service
metadata:
  name: coo-llm
spec:
  selector:
    app: coo-llm
  ports:
  - port: 80
    targetPort: 2906
  type: ClusterIP

ConfigMap:

apiVersion: v1
kind: ConfigMap
metadata:
  name: coo-llm-config
data:
  config.yaml: |
    version: "1.0"
    server:
      listen: ":2906"
    storage:
      runtime:
        type: redis
        addr: "${REDIS_ADDR}"
    llm_providers:
      - id: openai
        type: openai
        api_keys: ["${OPENAI_API_KEY}"]
        model: gpt-4o
    model_aliases:
      gpt-4o: openai:gpt-4o

Networkingâ€‹

Load Balancingâ€‹

nginx:

upstream coo-llm {
    server coo-llm-1:2906;
    server coo-llm-2:2906;
    server coo-llm-3:2906;
}

server {
    listen 80;
    server_name api.example.com;

    location / {
        proxy_pass http://coo-llm;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}

AWS ALB:

  • Target Group: EC2 instances or ECS tasks
  • Health Check: GET /health
  • SSL Termination: ACM certificate

Securityâ€‹

TLS Configuration:

server:
  listen: ":8443"
  tls:
    cert_file: "/etc/ssl/certs/coo-llm.crt"
    key_file: "/etc/ssl/private/coo-llm.key"

nginx with TLS:

server {
    listen 443 ssl;
    server_name api.example.com;

    ssl_certificate /etc/ssl/certs/api.crt;
    ssl_certificate_key /etc/ssl/private/api.key;

    location / {
        proxy_pass http://coo-llm;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

Monitoringâ€‹

Metricsâ€‹

Prometheus metrics are available at /metrics when enabled:

logging:
  prometheus:
    enabled: true
    endpoint: "/metrics"

Prometheus config:

scrape_configs:
  - job_name: 'coo-llm'
    static_configs:
      - targets: ['coo-llm:2906']
    metrics_path: '/metrics'

Loggingâ€‹

File logging:

logging:
  file:
    enabled: true
    path: "./logs/llm.log"
    max_size_mb: 100
    max_backups: 5

Logs include:

  • Request details (model, tokens, latency)
  • Provider selection
  • Errors and retries
  • Usage metrics

Loggingâ€‹

Centralized Logging:

logging:
  providers:
    - name: "elasticsearch"
      type: "http"
      endpoint: "https://es.example.com/_bulk"

Log Aggregation:

  • ELK Stack (Elasticsearch, Logstash, Kibana)
  • Splunk
  • Datadog
  • CloudWatch Logs

Scalingâ€‹

Multiple API Keysâ€‹

Add multiple keys per provider for higher throughput:

llm_providers:
  - id: "openai-prod"
    type: "openai"
    api_keys: ["${OPENAI_KEY_1}", "${OPENAI_KEY_2}", "${OPENAI_KEY_3}"]
    limits:
      req_per_min: 200  # Per key
      tokens_per_min: 100000

Redis for Productionâ€‹

Use Redis for persistent metrics and caching:

storage:
  runtime:
    type: "redis"
    addr: "redis:6379"
    password: "${REDIS_PASSWORD}"

Troubleshootingâ€‹

Common Issuesâ€‹

Application won't start:

# Check config syntax
./coo-llm -config configs/config.yaml

# Check environment variables
echo $OPENAI_API_KEY

Requests failing:

# Test basic connectivity
curl -X POST http://localhost:2906/v1/chat/completions \
  -H "Authorization: Bearer test-key" \
  -d '{"model": "gpt-4o", "messages": [{"role": "user", "content": "Hello"}]}'

# Check logs
tail -f logs/llm.log

Rate limiting errors:

  • Check your API key limits with the provider
  • Add more API keys to your configuration
  • Reduce request frequency

Logsâ€‹

Log files are written to ./logs/llm.log by default:

logging:
  file:
    enabled: true
    path: "./logs/llm.log"
    max_size_mb: 100
    max_backups: 5

Log entries include:

  • Request timestamps and models
  • Token usage and latency
  • Provider selection decisions
  • Errors and retry attempts

Securityâ€‹

API Key Managementâ€‹

Store API keys securely:

# Environment variables (recommended)
export OPENAI_API_KEY="sk-your-secure-key"

# Or use a secrets file (not in repo)
echo "OPENAI_API_KEY=sk-your-key" > .env

Client authentication:

api_keys:
  - key: "production-key"
    allowed_providers: ["openai-prod", "gemini-prod"]
    description: "Production client"

Best Practicesâ€‹

  • Use different API keys for different environments
  • Rotate keys regularly
  • Monitor usage patterns
  • Use HTTPS in production
  • Limit API key permissions with allowed_providers
